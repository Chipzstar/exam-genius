# AGENTS.md

## Cursor Cloud specific instructions

### Overview

ExamGenius is a Turborepo monorepo with two Next.js 15 apps and shared libs. See `README.md` for full architecture and scripts.

| App | Port | Notes |
|-----|------|-------|
| `dashboard-app` | 3000 | Requires Clerk/Stripe/OpenAI keys to fully function |
| `landing-page` | 4201 | Works standalone with no external services |

### Running services

- **Landing page**: `pnpm dev:landing` — works out of the box, no env vars needed.
- **Dashboard**: `pnpm dev:dashboard` — requires `.env.local` in `apps/dashboard-app/` with valid Clerk, Stripe, and OpenAI keys. Use `SKIP_ENV_VALIDATION=1` in that file to bypass Zod env checks during development without real keys.
- The `with-env` script in dashboard uses `dotenv-cli` (installed as a root devDependency) to load `.env.local`.

### Linting & testing

- `pnpm lint` — runs ESLint on all packages (via Turbo).
- `pnpm test` — runs Jest across the monorepo. Note: `shared-utils` has no test files so will fail without `--passWithNoTests`. Some existing tests have pre-existing failures (Mantine provider missing, lottie-web canvas issues, ESM transform gaps).

### Prisma

- After schema changes under `prisma/schemas/`, run `pnpm prisma:generate` from the repo root (merges schemas via prismerge, then generates client to `libs/shared/prisma`).
- The generated `prisma/schema.prisma` is auto-generated by prismerge — do not edit directly.
- Database operations (`db push`, `db migrate`, `db studio`) require a valid `DIRECT_DATABASE_URL` in the dashboard's `.env.local`.

### Gotchas

- The `pnpm.onlyBuiltDependencies` field in root `package.json` allowlists critical native packages (`prisma`, `@swc/core`, `sharp`, `unrs-resolver`, `@prisma/engines`). Without this, pnpm 10 blocks their postinstall scripts and the Prisma client / SWC compilation will fail.
- `dotenv-cli` is a root devDependency required for the dashboard's `with-env` script. If you see `dotenv: not found`, re-run `pnpm install`.
- The Jest config references `@nx/jest`, `@nx/react`, and `@nx/next` (legacy from Nx migration). These are installed as devDependencies to keep tests runnable.
- The dashboard app returns HTTP 500 without valid Clerk keys because Clerk middleware rejects requests with invalid publishable keys. This is expected in local dev without real credentials.
